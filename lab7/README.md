# Лабораторна робота: Робота з колекціями в мові Java (Collections Framework)

Цей проєкт є консольним Java-додатком, призначеним для вивчення, аналізу та практичного застосування динамічних структур даних з Java Collections Framework. Проєкт демонструє роботу зі списками об'єктів (`ArrayList`) та асоціативними картами ключ-значення (`HashMap`), реалізує механізми фільтрації, модифікації об'єктів всередині колекцій, а також взаємодію між асоційованими сутностями.

## 1. Структура проєкту

Проєкт спроєктовано із розподілом логічних блоків за окремими класами:

* TaskRunner.java — Головна точка входу в програму. Містить метод `main`, який керує створенням початкових даних, ініціалізує динамічний список об'єктів, виконує послідовну обробку масивів даних за заданими алгоритмами та запускає сервіс телефонної книги.
* Driver.java — Клас-модель, що описує сутність водія. Містить інформацію про ім'я, вік та стаж роботи водія.
* TaxiCar.java — Клас-модель, що описує сутність автомобіля таксі. Включає в себе характеристики марки (бренд), потужності двигуна, вартості, року випуску, а також містить посилання на об'єкт класу `Driver` (агрегація сутностей).
* TaxiHelper.java — Утилітарний сервісний клас для управління парком автомобілів. Реалізує логіку вибору випадкового автомобіля зі списку за допомогою класу `Random`, логує процес відправки авто за користувачем та фіксує його прибуття.
* PhoneBookService.java — Окремий сервісний клас, що реалізує бізнес-логіку телефонної книги. Демонструє використання та функціонування колекції `HashMap` та основних операцій над її елементами.

## 2. Ключові операції та алгоритми

### Завдання 1: Динамічні списки (ArrayList)
На прикладі обробки парку автомобілів таксі (`ArrayList<TaxiCar>`) реалізовано:
* Модифікацію полів об'єктів: підвищення потужності двигуна на 10% для першої половини списку та заміна пов'язаних об'єктів `Driver`.
* Покрокову зміну значень (потужність +10%, ціна +5%) для кожного другого елемента колекції за допомогою циклу з кроком `i += 2`.
* Умовну фільтрацію: перевірка умов стажу водія (якщо стаж менше 5 років, а вік перевищує 25 років, водій направляється на курси підвищення кваліфікації).
* Випадковий вибір елемента колекції на основі генератора індексів `Random.nextInt(fleet.size())`.

### Завдання 2: Асоціативні карти (HashMap)
У класі `PhoneBookService` виконано практичне тестування базових методів інтерфейсу `Map` на базі колекції з щонайменше 10 записів (Прізвище -> Номер телефону):
1. `put(K key, V value)` — додавання елементів.
2. `get(Object key)` — пошук значення за ключем.
3. `remove(Object key)` — видалення запису за ключем.
4. `containsKey(Object key)` — перевірка наявності заданого прізвища (ключа).
5. `containsValue(Object value)` — перевірка наявності заданого номера телефону (значення).
6. `size()` — отримання поточної кількості записів у колекції.
* Ітерація по карті та виведення її вмісту реалізовано через цикл `for-each` із використанням об'єкта `Map.Entry<String, String>`.
