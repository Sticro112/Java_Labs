# Лабораторна робота: Декомпозиція задач та робота зі статичними методами в Java

Цей проєкт є консольним Java-додатком, розробленим для вивчення принципів модульного програмування, процедурної декомпозиції, створення та виклику статичних методів із параметрами та значеннями, що повертаються, а також повторного використання утилітарних класів для валідації консольного введення.

## 1. Структура проєкту

Код розподілено між окремими класами з дотриманням принципу єдиної відповідальності (SRP):

* Main.java — Головна точка входу в програму. Керує життєвим циклом загального об'єкта Scanner та послідовно запускає демонстраційні та тестові методи кожного завдання.
* safeEngine.java — Утилітарний допоміжний клас, що містить статичні методи для безпечного зчитування числових типів даних (int, double) з потоку введення. Захищає програму від падіння при помилковому введенні тексту.
* Task1.java — Демонстрація роботи методу, що повертає значення. Містить метод calculateSquareArea(double side) для обчислення площі квадрата, а також окремий тестовий метод test1() для автоматичної перевірки роботи обчислювальної логіки в циклі від 0 до 10.
* Task2.java — Реалізація методу пошуку мінімального значення findMin(double a, double b, double c) серед трьох аргументів, що передаються за параметрами. Метод логує вхідні змінні та повертає найменше з них.
* Task3.java — Демонстрація передачі об'єктів (масивів) у методи. Містить метод printArray(int[] array), який приймає одновимірний масив цілих чисел та виводить усі його елементи в один рядок. Розмір та наповнення масиву динамічно задає користувач.
* Task4.java — Реалізація алгоритму пошуку екстремуму в масиві через окремий метод findMax(int[] array). Метод приймає масив довільної довжини, знаходить найбільше ціле число та повертає його як результат для подальшого виведення в консоль.

## 2. Механізм безпечного зчитування даних (safeEngine)

Взаємодія з користувачем через консоль захищена від виняткових ситуацій типу InputMismatchException (виникає, якщо замість очікуваного числа ввести текстовий рядок). Обробка реалізована за допомогою конструкцій try-catch всередині нескінченних циклів:
* readDouble(Scanner scan, String prompt) — циклічно запитує дробове число типу double (підтримує локалізацію, де роздільником є кома).
* readInt(Scanner scan, String prompt) — забезпечує аналогічний циклічний запит для зчитування цілих чисел типу int.

## 3. Компіляція та запуск

Для збирання проєкту та виконання байт-коду необхідна встановлена версія JDK (Java Development Kit) 11 або вище.

Послідовність команд у консолі термінала:
1. Компіляція всіх сирцевих файлів Java у поточній директорії:
   javac *.java

2. Запуск головного класу програми:
   java Main
