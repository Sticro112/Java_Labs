# Лабораторна робота: Створення графічного інтерфейсу користувача (GUI) за допомогою бібліотеки Swing

Цей проєкт є настільним (desktop) Java-додатком, розробленим для вивчення, аналізу та практичного застосування засобів побудови графічного інтерфейсу користувача. Програма реалізує калькулятор для розрахунку сили пружини (закон Гука) з використанням архітектурного шаблону проектування, розділеного на логіку представлення (інтерфейс) та бізнес-модель обчислень.

## 1. Архітектура та структура проєкту

Проєкт спроєктовано із жорстким розподілом обов'язків між класами для забезпечення модульності та масштабованості коду:

* TaskRunner.java — Точка входу в програму. Містить метод `main`, який використовує потокобезпечний механізм `SwingUtilities.invokeLater` для коректної ініціалізації та відображення графічного вікна всередині головного потоку обробки подій диспетчеризації (`Event Dispatch Thread`).
* SpringForceModel.java — Клас-модель обчислень. Відповідає суто за фізико-математичну бізнес-логіку програми. Містить статичний метод `calculateForce(double k, double x)`, який обчислює силу за формулою $F = k \cdot x$. Також реалізує валідацію вхідних параметрів: у разі передачі від'ємних значень генерує виняток `IllegalArgumentException`.
* SpringForceFrame.java — Клас представлення графічного інтерфейсу, що розширює базовий контейнер `JFrame`. Реалізує створення візуальних компонентів, їхнє абсолютне позиціонування на формі та реєстрацію слухачів подій для обробки дій користувача.

## 2. Графічні компоненти та обробка подій

Графічний інтерфейс побудовано на базі контейнерів та елементів управління бібліотеки `javax.swing`:
* `JTextField` — Текстові поля для введення коефіцієнта жорсткості ($k$), величини стиснення ($x$) та виведення фінального результату ($F$). Поле результату захищене від випадкового редагування користувачем.
* `JLabel` — Текстові підписи для маркування призначення відповідних полів введення.
* `JButton` — Кнопка ініціалізації розрахунку.

### Обробка подій та валідація даних:
Взаємодія користувача з інтерфейсом реалізована через підключення слухача `ActionListener` до об'єкта кнопки. Метод `actionPerformed` виконує комплексний захист від некоректних дій:
1. **Обробка порожніх або текстових значень:** Перехоплює `NumberFormatException`, якщо користувач ввів літери або залишив поля порожніми. Відображає діалогове вікно помилки `JOptionPane.showMessageDialog`.
2. **Логічна валідація:** Перехоплює `IllegalArgumentException` від моделі обчислень у разі спроби порахувати силу для від'ємної жорсткості чи стиснення.
3. **Форматування результату:** Отримане значення сили виводиться на екран із фіксованою точністю до 4 знаків після коми.
