# Лабораторна робота: Одновимірні масиви в мові Java

Цей проєкт є консольним Java-додатком, розробленим для вивчення, аналізу та практичного застосування одновимірних масивів, базових алгоритмів їх заповнення, обходу, фільтрації та модифікації елементів, а також механізмів безпечного введення числових даних.

## 1. Структура проєкту

Проєкт реалізовано із розподілом логічних блоків за окремими класами відповідно до принципу єдиної відповідальності (SRP):

* Main.java — Головна точка входу в програму. Координує послідовний виклик та виконання усіх завдань лабораторної роботи, керує життєвим циклом об'єкта Scanner.
* safeEngine.java — Утилітарний клас, який містить статичні методи для безпечного зчитування цілих та дробових чисел з консолі. Захищає програму від критичних збоїв при отриманні некоректних символів.
* Task1.java — Створення та ініціалізація двох окремих масивів розміром 50 елементів кожен. Перший масив автоматично заповнюється послідовністю парних чисел (від 2 до 100), другий — послідовністю непарних чисел (від 1 до 99). Результати виводяться в консоль за допомогою Arrays.toString().
* Task2.java — Демонстрація різних методів обходу статичного масиву з 10 елементів: виведення всіх елементів через цикли for та while, виведення елементів на непарних індексах, виведення елементів на парних індексах, а також повний обхід масиву у зворотному порядку.
* Task3.java — Динамічне створення масиву дробових чисел (double), розмір якого та значення кожного елемента вводяться користувачем вручну через консоль. Обчислюється та виводиться сума всіх елементів масиву.
* Task4.java — Робота з динамічним масивом цілих чисел (int). Програма зчитує елементи з консолі, виконує перевірку кожного значення на парність і змінює знак (значення на протилежне) для всіх знайдених непарних чисел, після чого виводить оновлений масив.

## 2. Механізм валідації вхідних даних (safeEngine)

Для запобігання винятковим ситуаціям типу InputMismatchException, які виникають при спробі зчитати текст замість очікуваного числа, утилітарні методи використовують блоки перехоплення винятків try-catch всередині нескінченних циклів:
* readDouble(Scanner scan, String prompt) — зчитує дробові числа типу double. Враховує регіональні налаштування операційної системи, де роздільником за замовчуванням є кома.
* readInt(Scanner scan, String prompt) — забезпечує аналогічний циклічний запит для коректного введення цілих чисел типу int.

## 3. Компіляція та запуск програми

Для збирання та запуску додатка необхідний встановлений пакет розробника JDK (Java Development Kit) версії 11 або вище.

Виконання команд через термінал:
1. Компіляція всіх сирцевих файлів проєкту:
   javac *.java

2. Запуск скомпільованого байт-коду програми:
   java Main
